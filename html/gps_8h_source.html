00049 /*
00050  * The structure describing an uncertainty volume in kinematic space.
00051  * This is what GPSes are meant to produce; all the other info is
00052  * technical impedimenta.
00053  *
00054  * All double values use NAN to indicate data not available.
00055  *
00056  * Usually all the information in this structure was considered valid
00057  * by the GPS at the time of update.  This will be so if you are using
00058  * a GPS chipset that speaks SiRF binary, Garmin binary, or Zodiac binary.
00059  * This covers over 80% of GPS products in early 2005.
00060  *
00061  * If you are using a chipset that speaks NMEA, this structure is updated
00062  * in bits by GPRMC (lat/lon, track, speed), GPGGA (alt, climb), GPGLL
00063  * (lat/lon), and GPGSA (eph, epv).  Most NMEA GPSes take a single fix
00064  * at the beginning of a 1-second cycle and report the same timestamp in
00065  * GPRMC, GPGGA, and GPGLL; for these, all info is guaranteed correctly
00066  * synced to the time member, but you'll get different stages of the same
00067  * update depending on where in the cycle you poll.  A very few GPSes,
00068  * like the Garmin 48, take a new fix before more than one of of
00069  * GPRMC/GPGGA/GPGLL during a single cycle; thus, they may have different
00070  * timestamps and some data in this structure can be up to 1 cycle (usually
00071  * 1 second) older than the fix time.
00072  *
00073  * Error estimates are at 95% confidence.
00074  */
00075 typedef double timestamp_t;     /* Unix time in seconds with fractional part */
